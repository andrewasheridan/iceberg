# 0007. Minimum Supported Python Version is 3.14

Date: 2026-03-21
Status: Accepted

## Context

`sheridan-iceberg` is a developer tool, not a library targeting broad compatibility. As a tool consumed directly by developers in their own environments, it is not constrained by the need to support the widest possible range of Python versions.

Python 3.14 introduces and stabilizes several language features that improve code clarity: structural pattern matching (`match`/`case`), the `type` statement for type aliases, and `X | Y` union syntax in type annotations without importing `Union` from `typing`.

## Decision

The minimum supported Python version is 3.14.

## Consequences

- Code may freely use `match`/`case`, `type` aliases, and `X | Y` union syntax without compatibility shims or imports.
- Users must have Python 3.14 installed. Since `iceberg` targets developers actively maintaining Python projects, this is an acceptable constraint.
- CI pipelines test against Python 3.14 only; there is no multi-version test matrix.
- This decision is revisited when a new Python release offers compelling features or when adoption patterns shift.
